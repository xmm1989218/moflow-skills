## Documentation Writing Guidelines

### Structure
- Start with a brief overview/purpose
- Use headings for logical sections
- Include code examples where applicable
- Add a table of contents for long documents

### Style
- Use present tense, imperative mood for instructions
- Be concise but complete
- Prefer examples over abstract descriptions
- Use consistent terminology throughout

### API Documentation
- Document all public interfaces
- Include request/response examples
- Specify parameter types and constraints
- Document error cases and status codes

### README
- Project name and one-line description
- Installation instructions
- Quick start example
- Configuration options
- License
